Glossary of Terms and GUI ElementsÂ¶

acceleration property
This is a speed threshold related property. When you want it to trigger on crossing the threshold whilst accelerating, set this to true. False means that it will trigger whenever you are above the speed threshold. This is a boolean value.

Note

ACC:1 - This will enable this property

area
This is a shape which represents 1 or more media files. They can be seen in the map view.
area list
Found at the bottom of the right column in the GUI. This is a list of areas and the domains in which they can be found. Use this to go to a specific area (double-click on the name) as well as select an area to edit.
area property view
Found at the bottom of the left column in the GUI This displays all of an areas properties in a list. You should find a definition for each of the properties in the glossary. This is also used to change properties quickly using a specially formatted string, similar to that used in the MapSource comment field. It is also known as the coded property string. However, this only allows editing of, mostly, area related properties. This means fewer options are changeable/necessary here.
boolean
This is a true or false value. In properties it is usually specified as 1 (which is true) and 0 (which is false).
can re-trigger threshold property
This allows speed threshold triggers to re-trigger without having to leave the area itself. This is a boolean value.

Note

STCR:1 - this will enable re-triggering.

Warning

Please note that for this to work correctly, the maximum trigger count property has to be set to a sufficiently high level.

coded property string
This allows the changing of an areas properties using just one continuous line of text. Each property is separated by a single plus ‘+’ character. An example would be AR:50+D:260, which is, of course, two different properties. Each property is divided into two segments by the colon ‘:’ character. The first segment is the property name. The second segment is the value to set the property to. Unless otherwise specified, the properties only apply to the first media/direction. The following properties are supported:
  • user play prompt property
  • radius property
  • sequence area property
  • content base path property
  • directions property
  • delay time property
  • priority property
  • reset played areas property
  • maximum trigger count property
  • speed above threshold property
  • speed below threshold property
  • can re-trigger threshold property
  • trigger type property
  • days of week property
  • acceleration property
  • deceleration property
  • time window property
  • date window property
content base path property
This is the first part of the path used to store media files. An example would be FirstPart\Second\Third. ‘FirstPart’ is what this property is. This is usually the name of the domain

Note

CBP:RootDirectory

date window property
This specifies the start and ending dates in-between which the media may play. The two dates are separated using a dash ‘-‘. The year, month and day are separated using a forward slash ‘/’. There are two different types of date ranges that can be entered.
  1. One with a year, specifies a specific year in which to trigger.
  2. One without a year, specifies a seasonal date range. This means that every year, between the specified month/day window, the media will be played.

Note

Z:1999/12/24-2000/12/23 - this specifies a specific year during which the media may play. Z:1/1-3/1 - this means that, every year, only the first two months will be valid trigger periods.

days of week property
This allows you to specify on which days the media may play. Just use the first three characters of the day you want. Each day is comma separated.

Note

W:MON,TUE,WED - this will only trigger the media on Monday, Tuesday and Wednesday

deceleration property
This is a speed threshold related property. When you want it to trigger on crossing the threshold whilst decelerating, set this to true. False means that it will trigger whenever you are below the speed threshold. This is a boolean value.

Note

DEC:1 - This will enable this property

delay time property
This property is the time in seconds with which to delay the start of playing this media file upon entering the area.

Note

DT:180. This will delay the playing of the media by 3 minutes.

directions property
This is used to set the heading, file name and variance. Any one of the successive options can be left out. More than one direction is specified by separating them with a pipe ‘|’.

Note

D:150,MyFile.mp3.45|D:260,AnotherFile.mp3 - this means the first media file ‘MyFile.mp3’ will only play when heading is 150 degrees with a variance of 45 degrees. The second media file ‘AnotherFile.mp3’ will play in the heading of 260. The default variance is 45.

domain
This is a large shape that acts as a container for several area‘s. Its name is used to point to the first directory of where all media files relating to it can be found
GPS
This is a.k.a. global positioning system. It is used to pinpoint locations using satellite.
GUI
This is a.k.a. general user interface. It is what the user interacts with when they use the application.
map view
Found in the centre of the GUI. This is the main purpose of the application, the visualisation of GPS positioned media files.
maximum trigger count property
This property specifies the maximum number of times the media can be triggered or played. It could also be called the maximum play count property.

Note

The default is 1. Example: S:999 - this is usually a good enough number if you don’t want restrictions.

media file
This can only be audio (*.mp3) at present, but other types will be possible in the future.
  • Audio
  • Video
  • Image
  • Text
  • Menu
priority property
This property assigns one of three priority levels to the media. The most important is ‘1’, the least is ‘3’. Example: PR:2 - which is also the default priority level.
query
A query is a stored filter. It will find specific information, based on the criteria given by you.
query editor
Found at the top of the right column. This editor allows editing of queries which can then be used to filter the list of areas displayed in the area list.
query list
Found below the query editor in the right column. This is a list of queries that have been created by you. Select any one of these queries to filter the area list.

Note

The default query is created automatically for you and will allow you to see all areas available to you, across all domains.

user play prompt property
This determines whether the user will be prompted before playing back the media. This is a boolean value.

Note

A:1 - which means the user will be prompted before playing the media.

radius property
This is the radius, in meters, of a circular area.

Note

AR:2500 - this specifies a circular area with a radius of 2.5km or 2500m

reset played areas property
This says that when you enter this area, all areas that have been marked as played will have their play count reset. This is a boolean value.

Note

R:1 - which means, yes, reset played areas.

sequence area property
This refers to the parent area. This will allow chaining of media. If the parent has not been played, this area will not play either.

Note

C:MyParent - where MyParent is the name of the area to which this child area is refering.

speed above threshold property
This property specifies the speed above which the speed threshold will trigger. The unit of measurement is km/h.

Note

SAT:250 - this refers to speeds above 250km/h

speed below threshold property
This property specifies the speed below which the speed threshold will trigger. The unit of measurement is km/h.

Note

SBT:900 - this refers to all speeds below 900km/h

text box
This is a rectangular box used for text entry in the GUI
time window property
This sets the start and end time during which the media may play. The two times are separated using a dash ‘-‘. The hours and minutes are separated using a period ‘.’.

Note

X:15.30-22.59 - The start time is 3:30pm and the end time is 10.59pm

trigger type property
This specifies the type of trigger. There are 4 types, each specified using an alphanumeric character.
  • A - Always play the media when entering the area.
  • E - Only trigger on entering the area.
  • X - Trigger only on exiting the area.
  • W - Only trigger the playing of the media, while you are in the area.

Note

T:A - this is also the default
